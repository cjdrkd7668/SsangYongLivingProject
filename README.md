쌍용리빙 프로젝트
=============
Spring Framework를 사용한 MVC 패턴 웹 사이트 프로젝트
-------------

### 개발환경

* 사용 언어
  * Java, JDK 1.8.0_221
  * HTML, CSS, JavaScript
  * SQL, PL/SQL
* 개발 도구
  * Eclipse IDE for Enterprise Java Developers
  * Visual Studio Code Version 1.41
  * Oracle SQL Developer
* 애플리케이션 프레임워크
  * Spring Framework 4.3.8
* 미들웨어 솔루션
  * DB : Oracle DataBase
  * WAS : Apache Tomcat 8.5v
* 형상 관리 도구 : Github Desktop

<hr>

### 목적
 * 자취하는 인구에게 유용할법한 서비스를 제공하는 웹 어플리케이션 개발.

### 후기
이청강
>스프링 프레임워크를 처음 사용해본 프로젝트라 초기 프로젝트의 환경설정을 하는데에 어려움을 겪었습니다.
하지만 이전 프로젝트와 비교해보았을 때 비교적 정형화된 코드 작성이 가능했고, 이 때문에
팀원간의 커뮤니케이션이 원활할 수 있었습니다. 특히, 타일즈 프레임워크를 사용하여 웹 페이지의 템플릿을 구성하는 것이
유용했고, 매퍼 인터페이스를 사용하여 데이터베이스와 연결, SQL을 호출할 수 있어서 효율적으로
데이터를 가져오는 한편 팀원 모두가 작성한 매핑 정보를 패키지 하나로 접근하여 볼 수 있어서 편리했습니다.
아쉬운 점은 이번 프로젝트가 교육센터의 수료와 개인적인 포트폴리오 준비와 맞물려서  충분한 시간을 할애하지 못했다는 것입니다.
프로젝트 개발에 시간을 얼마나 분배할 수 있는 지 예측하는 것이 중요하다는 것을 다시금 느꼈습니다.

김소리
>spring framework를 이용하여 프로젝트를 진행하며
의존주입, myBatis, Interface의 사용, Open api 사용에 
조금 더 익숙해지는 계기가 되었습니다. 
또한, spring은 환경을 설정하는 것이 매우 중요한만큼,
개발환경 setting을 무리없이 할 수 있게끔 숙달되는 것이 
꼭 필요함을 알 수 있었습니다.
>
>한편 이번 spring 프로젝트는 교육과정 수료와 맞물려 타 프로젝트에 비해
상대적으로 아주 짧은 개발 기간동안 진행되었습니다.
교육과정 절차를 잘 마무리짓는 것 만큼이나, 
마지막 프로젝트 또한 일정에 맞게끔 완수될 수 있도록 
시간 분배를 철저히 하여 팀프로젝트의 완성도를 높이는 것이
중요함을 깨달을 수 있었습니다.

이찬미
>스프링 프레임워크를 사용하면서 선생님 말씀대로 기본 환경 설정하는 것이 가장 힘들었습니다. 팀원들이 모두 같은 개발 환경이 아니었기 때문에 좀 더 신경써야 할 부분이 많았는데 그 점이 아쉽습니다.
DI, AOP를 이용하면서 바로 앞선 프로젝트보다 코드가 간결해졌고 인터페이스를 사용했기 때문에 결합도가 낮아져 코드 수정하기가 쉬웠습니다.
자유자재로 다룰 수 있지는 않지만 게시판과 네이버 API 검색 기능을 사용하면서 패턴을 익혔습니다.
가장 적은 인원으로 프로젝트를 했는데 커뮤니케이션도 잘 되었고 문제 없이 프로젝트를 끝내 좋았습니다.
